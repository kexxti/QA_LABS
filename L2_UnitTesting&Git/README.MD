# Ход работы

## 1. Описание найденного дефекта

В ходе тестирования метода `pop_back()` в контейнере `s21::list` была намеренно добавлена ошибка в реализацию метода.

Исходная корректная реализация:

```cpp
void pop_back() {
    if (empty()) {
        throw std::logic_error("List is empty");
    }

    if (head_ == tail_) {
        delete head_;
        head_ = nullptr;
        tail_ = nullptr;
        --size_;
        return;
    }

    tail_ = tail_->pre_;
    tail_->next_ = nullptr;

    --size_;
}
```

Для демонстрации процесса тестирования была создана ошибочная версия метода:
```cpp
void pop_back() {
    if (empty()) {
        throw std::logic_error("List is empty");
    }

    if (head_ == tail_) {
        delete head_;
        head_ = nullptr;
        tail_ = nullptr;
        --size_;
        return;
    }

    tail_ = tail_->pre_;
    tail_->next_ = nullptr;

    // размер списка не уменьшается
}
```
В данной версии метода после удаления элемента размер контейнера остается прежним, что приводит к некорректной работе метода size().


## 2. Покрытие дефекта тестом

Для обнаружения ошибки был написан следующий модульный тест:
```cpp
TEST(ListTest, PopBackDoesNotDecreaseSize) {
    s21::list<int> l;

    l.push_back(1);
    l.push_back(2);
    l.push_back(3);

    l.pop_back();

    EXPECT_EQ(l.size(), 2);
}
```
При запуске тестов был получен следующий результат:

```
Expected equality of these values:
  l.size()
    Which is: 3
  2
```
Тест показал, что после удаления элемента размер списка не изменился, хотя должен был уменьшиться на 1.

Для исправления ошибки в метод pop_back() была возвращена строка (в исходном коде используется условная компиляция, для переключения между корректной и некорректной реализациями):

```
--size_;
```

## Описание дефекта

### Шаги воспроизведения

1. Создать объект `s21::list<int>`
2. Добавить несколько элементов
3. Вызвать `pop_back()`
4. Проверить размер списка через `size()`

Пример:

```cpp
s21::list<int> l;

l.push_back(1);
l.push_back(2);
l.push_back(3);

l.pop_back();

std::cout << l.size();
```

### Фактический результат

После удаления одного элемента метод size() возвращает значение 3, хотя в списке осталось только два элемента.

### Ожидаемый результат

После удаления одного элемента метод size() должен возвращать значение 2.
