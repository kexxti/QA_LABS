# Ход работы

## Задание 1

Для выполнения лабораторной работы был выбран один из проектов Школы 21 — реализация игры «Тетрис» на языке C с использованием конечного автомата (FSM).

 | Состояние  | Описание                              |
 |------------|---------------------------------------|
 | Begin      | Ожидание нажатия кнопки «Старт»       |
 | Spawn      | Создание новой фигуры                 |
 | Moving     | Обработка ввода, движение фигуры      |
 | Shifting   | Автоматическое падение фигуры         |
 | Attaching  | Присоединение фигуры к полю           |
 | GameOver   | Конец игры                            |
 | Break      | Пауза                                 |

 ![Граф FSM](.//BrickGame_v1/misc/images/fsm.png)

Следует отметить, что в проекте отсутствует объектно-ориентированное программирование в классическом понимании, однако структура проекта организована таким образом, что хорошо подходит для документирования и построения иерархий данных.

В проекте используются несколько взаимосвязанных структур данных:

- `FullInfo`
- `GameInfo_t`
- `Figure`

Также используются перечисления:

- `State`
- `UserAction_t`

Ниже представлена схема взаимосвязи структур данных.

![Иерархия структур](.//BrickGame_v1/misc/images/structs.png)

## Задания 4 и 5

Для всех структур данных, перечислений и публичных функций были написаны документирующие комментарии.

В комментариях использовались:

- описание назначения сущностей
- описание параметров через `@param`
- описание возвращаемых значений через `@return`

Также были добавлены обычные поясняющие комментарии в участках кода, требующих дополнительного объяснения.

В документацию были включены:

- математические формулы
- изображения
- сгенерированные графы

# Вывод

В ходе выполнения лабораторной работы был выбран проект реализации игры «Тетрис» на языке программирования C.

Для генерации технической документации был использован инструмент `Doxygen`, поддерживающий язык C.

Были написаны документирующие комментарии для структур данных, перечислений и публичных функций. В документацию добавлены математические формулы, изображения и автоматически сгенерированные графы связей.

После генерации HTML-документации были получены страницы с описанием структур проекта, графами зависимостей и навигацией по исходному коду.

В результате выполнения лабораторной работы были получены практические навыки документирования программного обеспечения и автоматической генерации технической документации.
